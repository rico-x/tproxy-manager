# TPROXY Manager v26.4.25.1

Изменения перечислены начиная с релиза `v26.4.24` / commit `71c21d78ce89f12f9e5c78d8b768060a45faada3`.

## Главное

- Пакет стал самодостаточным: в состав добавлен встроенный `vless2json.sh`, базовые Watchdog-шаблоны и все runtime-скрипты, необходимые для работы без внешних файлов проекта.
- Добавлена поддержка двух линий OpenWrt без SDK: `.ipk` для OpenWrt 24.10 / `opkg` и `.apk` для OpenWrt 25.12 / `apk`.
- Реализованы отдельные GitHub Pages feed-ветки `/24.10/` и `/25.12/` с индексами, ключами и release assets для обеих систем пакетов.
- Watchdog расширен до полноценного модуля управления VLESS-ссылками, шаблонами, проверками живости, ручным применением ссылок и автоматической ротацией.

## Watchdog

- Добавлена вкладка `WATCHDOG` в LuCI и отдельный сервис `/etc/init.d/tproxy-manager-watchdog`.
- Добавлена таблица VLESS-ссылок с комментариями, статусами `OK / Error / Не проверялась`, временем последней проверки и действиями `Применить`, `Проверить`, `Ред.`, `Удалить`, `Вверх`, `Вниз`.
- Добавлен массовый сворачиваемый редактор `LINKS_FILE` для удобной вставки списка ссылок.
- Добавлены режимы автоматического выбора ссылки: `по порядку` и `случайно`.
- Добавлено временное исключение нерабочих ссылок из автоматической ротации на заданный период.
- Добавлена фоновая проверка ссылок по настраиваемому таймеру.
- Добавлено ручное применение конкретной ссылки через отдельный test-instance без изменения рабочего Xray/Mihomo до успешной проверки.
- Добавлены отдельные редактируемые шаблоны: outbound-шаблон и test-template для проверки ссылок.
- Добавлена кнопка очистки Watchdog-лога.
- Runtime Watchdog разделён на внутренние shell-библиотеки в `/usr/libexec/tproxy-manager/watchdog/`, entrypoint оставлен тонким.

## LuCI и валидация

- Добавлен общий Lua utility layer для повторно используемых операций чтения/записи файлов, JSONC-проверки, shell escaping, state parsing и flash-сообщений.
- Убран дрейф defaults из LuCI: канонический источник начальных значений теперь `/etc/uci-defaults/90_tproxy_manager`.
- Зафиксирован единый default `nft_table=tp_mgr`.
- Усилена серверная валидация TPROXY-настроек, интерфейсов, numeric-полей и быстрого добавления DHCP IP.
- GEO-модуль больше не затирает конфиг пустым списком при невалидном JSONC.
- Добавлена серверная проверка cron-расписаний GEO.
- Mihomo-конфиг проверяется через `mihomo -t -f` перед записью на диск.
- Xray JSON/JSONC-редактор получил подсветку синтаксиса, статус валидации под редактором и сохранение текущего выбранного файла после нажатия `Сохранить`.

## Установка, defaults и данные

- При установке создаётся `/usr/share/tproxy-manager`.
- Если `/etc/tproxy-manager/geo-sources.conf` отсутствует или пустой, он заполняется источниками `GeoIP` и `GeoSite` от Loyalsoldier.
- В post-install добавлена рекомендация настроить proxy daemon на datadir `/usr/share/tproxy-manager/`, например для Xray: `uci set xray.config.datadir='/usr/share/tproxy-manager/'`.
- Добавлена рекомендация установить `https-dns-proxy`, `luci-app-https-dns-proxy` и `luci-i18n-https-dns-proxy-ru` для защиты DNS от утечек.
- Исправлены права исполняемых init.d и runtime-скриптов.

## Packaging и CI

- Убрана зависимость сборки от OpenWrt SDK для скриптового noarch-пакета.
- Добавлена быстрая сборка `.ipk` и `.apk` из общего payload-каталога без дублирования файлов.
- Исправлена генерация зависимостей `.apk`, чтобы `apk add tproxy-manager` подтягивал declared dependencies.
- Исправлена подпись `apk` feed и проверка `apk update` без `UNTRUSTED signature`.
- Добавлены smoke-проверки GitHub Actions: shell syntax, Lua syntax, shellcheck, `.ipk/.apk` metadata, feed indexes и landing pages.
- Исправлены проверки tar-based `.ipk` и имён `tproxy-manager_..._all.ipk`.

## Документация

- README переработан в пользовательский формат: описание проекта, установка, функции, рекомендации и диагностика.
- Добавлен дисклеймер о законном назначении проекта как средства администрирования локального роутера и управления трафиком.
- Инструкции установки разделены для OpenWrt 24.10 (`opkg/.ipk`) и OpenWrt 25.12 (`apk/.apk`).
- Скриншоты размещены по соответствующим разделам документации.
- Добавлена отдельная документация по встроенному `vless2json.sh`.
