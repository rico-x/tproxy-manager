# TPROXY Manager v26.5.22

## Главное

- Watchdog получил batch-проверку VLESS-ссылок: один test-instance проверяет пачку ссылок через отдельные outbound tags и локальные SOCKS-порты.
- Массовая проверка `check-all` больше не блокирует обычный watchdog-loop: для неё используется отдельный lock и отдельный диапазон портов.
- Добавлен режим выбора ссылки `самый быстрый`, который использует сохранённое время ответа живых ссылок.
- На вкладке Watchdog теперь видно активную ссылку или подписку: строка состояния показывает `ACTIVE`, а активные строки подсвечиваются в таблицах.

## Watchdog batch-check

- Добавлен внутренний shell-модуль `/usr/libexec/tproxy-manager/watchdog/batch.sh`.
- Добавлен базовый batch-template `/etc/tproxy-manager/watchdog-batch-test-config.template.jsonc`.
- В defaults добавлены параметры:
  - `watchdog_batch_check_enabled=1`
  - `watchdog_batch_check_port_start=10882`
  - `watchdog_batch_check_batch_size=64`
  - `watchdog_batch_check_concurrency=8`
  - `watchdog_batch_check_fallback=1`
  - `watchdog_batch_test_template_file=/etc/tproxy-manager/watchdog-batch-test-config.template.jsonc`
- Дефолтный batch-порт теперь начинается с `10882`, чтобы не конфликтовать с основным `watchdog_test_port=10881`.
- Если batch-template или test-instance не стартует, Watchdog может вернуться к индивидуальной проверке при включённом fallback.

## Состояние ссылок и выбор активной

- В state каждой ссылки сохраняется время ответа:
  - `LAST_REQUEST_TIME_MS`
  - `LAST_REQUEST_TIME_TEXT`
- Таблица VLESS-ссылок показывает статус в формате `OK · N ms` или `Error · timeout`.
- Режим `самый быстрый` сортирует живые ссылки по `LAST_REQUEST_TIME_MS`; если данных скорости ещё нет, используется fallback на порядок файла.
- Активная запись определяется по текущему `OUTBOUND_FILE`; если совпадение не найдено, используется fallback по `LAST_APPLIED_HASH`.

## LuCI

- В статусе Watchdog добавлено поле `ACTIVE`.
- Активная VLESS-ссылка получает бейдж `ACTIVE` и подсветку строки.
- Если активная ссылка пришла из подписки, соответствующая подписка также подсвечивается и получает бейдж `ACTIVE`.
- В настройках Watchdog добавлены поля batch-проверки и режим `самый быстрый`.
- Dirty-guard учитывает новые batch-поля.

## Runtime

- `check-all` и основной watchdog-loop разделены по lock, поэтому могут работать параллельно.
- Фоновая проверка ссылок запускает `check-all` асинхронно и не задерживает обычную проверку текущего proxy.
- Логирование Watchdog больше не пишет штатные сообщения в stderr, чтобы procd не помечал нормальные проверки как `daemon.err`.
- `vless2json.sh` получил опции `--one-outbound` и `--tag`, которые используются для генерации batch-outbound с уникальными тегами.
