# TPROXY Manager v26.5.23

## Главное

- Исправлена принудительная ротация Watchdog: кнопка теперь учитывает текущий режим `SELECTION_MODE`, выбранный в форме LuCI.
- Режим `самый быстрый` больше не заменяется старым сохранённым значением UCI, если пользователь выбрал его в форме и сразу нажал `Принудительная ротация`.

## Watchdog

- LuCI передаёт текущий `watchdog_selection_mode` в runtime при запуске `test-rotate`.
- Runtime принимает временный override через переменную окружения `WATCHDOG_SELECTION_MODE`.
- Автоматическая ротация и ручная принудительная ротация теперь используют одну и ту же логику выбора кандидатов: `по порядку`, `случайно` или `самый быстрый`.

## Диагностика

- В лог Watchdog добавлены строки:
  - `принудительная ротация: selection_mode=...`
  - `режим выбора ссылок: ...`
- По этим строкам можно быстро проверить, какой режим реально использовался при ротации.

## Проверка

- Локально пройдены `sh -n`, `luac -p`, `shellcheck -S error` и `git diff --check`.
- На тестовом роутере проверено, что override `WATCHDOG_SELECTION_MODE=fastest` применяется runtime-скриптом.
- Проверено, что fastest-сортировка первым выбирает ссылку с минимальным `LAST_REQUEST_TIME_MS`.
