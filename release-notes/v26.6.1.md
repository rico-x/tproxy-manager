## Changes

- Added browser-side Happ decrypt block on the Watchdog page.
- Supported Happ link formats: `happ://crypt/`, `crypt2`, `crypt3`, `crypt4`, `crypt5`.
- Added packaged static RSA key data for `crypt5`, so decrypt works without Node/npm runtime dependencies on the router.
- Combined `Happ capture` and `Happ decrypt` into one collapsed `Happ` block with separate inner sections.
- Removed the decrypt result copy button; results stay in a selectable output textarea.
- Updated README documentation for the unified Happ block and Happ decrypt behavior.
